## 核心概念

| 术语 | 英文 | 定义 |
|------|------|------|
| **CDN** | Content Delivery Network | 内容分发网络，通过分布式节点加速内容分发 |
| **隧道** | Tunnel | 在两个网络节点之间建立的加密数据传输通道 |
| **转发** | Forward/Relay | 将网络流量从一端传递到另一端 |
| **源站** | Origin Server | 内容的原始服务器，CDN 节点回源获取内容 |
| **边缘节点** | Edge Node | 部署在用户就近位置的 CDN 节点 |
| **主控节点** | Master Node | 负责管理和控制整个 CDN 系统的节点 |
| **被控节点** | Agent Node | 接受主控节点管理的边缘/核心节点 |

---

## 节点类型

| 术语 | 英文 | 定义 |
|------|------|------|
| **边缘节点** | Edge Node | 部署在边缘位置，直接接收用户请求的节点 |
| **核心节点** | Core Node | 部署在核心网络位置，负责转发到源站的节点 |
| **L2节点** | L2 Node | 二级中转节点，用于多层架构中的中间转发 |
| **Master** | Master Node | 主控节点，统一管理系统配置和监控 |
| **Agent** | Agent Node | 被控节点，执行具体转发任务 |

### 节点类型对比

| 类型 | 位置 | 主要职责 | 典型端口 |
|------|------|----------|----------|
| Edge | 香港/海外 | 接收客户端连接、TLS 卸载、协议转换 | 443 (QUIC), 8080 (WS) |
| Core | 大陆 | 转发到源站、认证、限流 | 8443 (QUIC), 8444 (WS) |
| Master | 任意 | 统一管理、配置下发、监控采集 | 8080 (HTTP), 50051 (gRPC) |

## 协议与技术

| 术语 | 英文 | 定义 |
|------|------|------|
| **QUIC** | Quick UDP Internet Connections | 基于 UDP 的传输层协议，支持多路复用 |
| **HTTP/3** | HyperText Transfer Protocol v3 | 使用 QUIC 作为传输层的 HTTP 协议 |
| **WebSocket** | WS | 双向实时通信协议 |
| **SSE** | Server-Sent Events | 服务器推送事件，单向实时通信 |
| **TLS** | Transport Layer Security | 传输层安全协议 |
| **gRPC** | Google Remote Procedure Call | 高性能 RPC 框架 |
| **gost** | GO Simple Tunnel | Go 语言实现的代理隧道工具 |

### 协议特性

| 协议 | 特点 | 适用场景 |
|------|------|----------|
| QUIC | 低延迟、0-RTT、多路复用 | 现代浏览器、移动应用 |
| WebSocket | 双向通信、长连接 | 实时聊天、协作工具 |
| SSE | 简单、服务器推送 | 消息通知、实时更新 |
| TLS | 加密传输、安全 | 所有 HTTPS 通信 |

## 安全相关

| 术语 | 英文 | 定义 |
|------|------|------|
| **CC攻击** | Challenge Collapsar | 应用层请求洪泛攻击，耗尽服务器资源 |
| **DDoS** | Distributed Denial of Service | 分布式拒绝服务攻击 |
| **5秒盾** | Five Second Shield | 高频访问限制防护机制 |
| **TLS终止** | TLS Termination | 在边缘节点解密 HTTPS，获取明文 HTTP |
| **IP白名单** | IP Whitelist | 允许访问的 IP 地址列表 |
| **IP黑名单** | IP Blacklist | 禁止访问的 IP 地址列表 |
| **限流** | Rate Limiting | 限制请求频率，防止过度访问 |
| **JW认证** | JWT Authentication | 基于 JSON Web Token 的认证方式 |

### 安全防护机制

| 机制 | 说明 |
|------|------|
| TLS 1.3 | 最新 TLS 版本，减少握手延迟 |
| 限流保护 | 防止 DDoS/CC 攻击 |
| 5秒盾 | 高频访问限制 |
| IP 白名单/黑名单 | 访问控制 |
| JWT 认证 | API 接口认证 |

## 运维相关

| 术语 | 英文 | 定义 |
|------|------|------|
| **健康检查** | Health Check | 检查节点和服务是否正常 |
| **故障转移** | Failover | 主节点故障时自动切换到备用节点 |
| **负载均衡** | Load Balancing | 将流量分配到多个节点 |
| **配置热更新** | Hot Reload | 不重启服务的情况下更新配置 |
| **优雅关闭** | Graceful Shutdown | 优雅地停止服务，不影响现有连接 |
| **滚动更新** | Rolling Update | 逐个更新节点，保持服务可用 |

### 运维操作

| 操作 | 说明 |
|------|------|
| 部署 | 安装和配置系统组件 |
| 监控 | 监控系统状态和性能指标 |
| 告警 | 当指标异常时发送通知 |
| 备份 | 定期备份数据和配置 |
| 恢复 | 从备份中恢复系统 |

## 性能指标

| 指标 | 定义 |
|------|------|
| **并发连接数** | 同时活跃的连接数量 |
| **QPS** | Queries Per Second，每秒请求数 |
| **P99延迟** | 99% 请求的响应时间低于该值 |
| **吞吐量** | 单位时间内传输的数据量 |
| **可用性** | 系统正常运行时间比例 |

### 性能目标

| 指标 | 目标值 |
|------|--------|
| 并发连接数 | 100,000+ / 节点 |
| QPS | 500,000+ / 全节点 |
| P99 延迟 | < 100ms |
| 可用性 | 99.9% |
